Invoice Creation API Documentation

Introduction

Welcome to the Invoice Creation API documentation. This API endpoint allows you to create invoices by providing customer details, product information, payment details, and other relevant data. The API is designed to be simple and intuitive, making it easy for you to integrate invoice creation functionality into your applications.

This documentation will guide you through the process of using the API, including the required request format, parameters, authentication, and example usage.

Base URL

All API requests are made to the following base URL:

https://your-api-domain.com/invoices

Authentication

To use the Invoice Creation API, you need to authenticate your requests using an API key. The API key should be included in the request headers as x-api-key.

Headers

{
  "Content-Type": "application/json",
  "x-api-key": "your_company_api_key"
}

Note: Replace your_company_api_key with the API key provided to you. If you don't have an API key, please contact the API administrator to obtain one.

Endpoint: Create a New Invoice

HTTP Method

POST

URL

https://your-api-domain.com/invoices

Description

Creates a new invoice with the provided customer details, products, payment information, and other optional data.

Request Body

The request body should be a JSON object containing the following fields:

Required Fields

  • customerName (string): The name of the customer.
  • products (array of objects): A list of products included in the invoice.
    • quantity (number): The quantity of the product.
    • price (number): The price per unit of the product.
    • productDescription (string): A description of the product.
    • productType (string): The type or category of the product.
  • paymentInfos (array of objects): Payment information for the invoice.
    • paymentMethod (string): The method of payment (e.g., "Credit Card", "Bank Transfer").
    • paymentType (string): The type of payment (e.g., "Online", "Offline").

Optional Fields

  • customerEmail (string): The email address of the customer.
  • customerAddress (string): The street address of the customer.
  • customerCity (string): The city of the customer.
  • customerState (string): The state or province of the customer.
  • customerCountry (string): The country of the customer.
  • customerPostCode (string): The postal code of the customer.
  • customerTaxId (string): The tax identification number of the customer.
  • discountAmount (number): The discount amount to be applied to the invoice.
  • gst (number): The GST (Goods and Services Tax) rate to be applied. Default is 18.
  • isInclusive (boolean): Indicates whether the prices are GST inclusive (true) or exclusive (false). Default is false.
  • credit (number): Any credit amount to be applied to the invoice.
  • createdAt (string): The date and time the invoice was created, in ISO 8601 format (e.g., "2023-10-10T12:34:56Z"). If not provided, the current date and time will be used.
  • paymentInfos.transactionId (string): The transaction ID for the payment. Optional.

Calculation Details

Amount Calculation:

The amount field represents the total amount including GST (if isInclusive is false) or excluding GST (if isInclusive is true).

  • When isInclusive is false: 
    amount = sum of (quantity × price) + GST
  • When isInclusive is true: 
    amount = sum of (quantity × price) (GST is included in the prices)

GST Calculation:

  • If isInclusive is false, GST is added to the base amount.
  • If isInclusive is true, GST is calculated from the base amount.

Total Calculation:

total = amount - discountAmount - credit

Error Handling

The API uses standard HTTP status codes to indicate the success or failure of an API request. In general:

  • 200 OK: The request was successful.
  • 201 Created: The resource was successfully created.
  • 400 Bad Request: The request was invalid or cannot be served. The exact error is provided in the error message.
  • 401 Unauthorized: Authentication failed or was not provided.
  • 403 Forbidden: The request is understood, but it has been refused or access is not allowed.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: An error occurred on the server.

Error Response Format

Errors are returned in the following JSON format:

{
  "error": "Error message detailing what went wrong."
}

Usage Notes

  • Date and Time Formats: All date and time values should be in ISO 8601 format.
  • Currency: All monetary values are in the currency specified by your company settings (e.g., INR for Indian Rupees). Ensure that you use the correct currency in your calculations.
  • GST Rates: The default GST rate is 18%. If your products are subject to a different GST rate, specify it in the gst field.
  • Authentication: Always include your API key in the request headers. Requests without a valid API key will be rejected.
  • Validation: Ensure that all required fields are provided and that they contain valid data types.
